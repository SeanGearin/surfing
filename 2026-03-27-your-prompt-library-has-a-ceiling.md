# Your Prompt Library Has a Ceiling

**Published:** Fri, 27 Mar 2026 13:18:38 GMT  
**Link:** https://surfing.substack.com/p/your-prompt-library-has-a-ceiling

---

@cyrilXBT just published one of the best AI systems breakdowns I’ve read. A prompt library inside Obsidian, connected to Claude Code. Versioned prompts treated like code. A daily workflow that turns raw research into published content in under an hour.

Read it before you read this. He earned that.

I want to offer something different — not a rebuttal. A next level.

What Cyril got exactly right

Chat windows are graveyards. Every session starts from zero. The best outputs you’ve ever gotten are buried somewhere you’ll never find them. Building a library of versioned, iterated prompts is real infrastructure — it compounds, it’s yours, and it’s meaningfully better than what almost everyone is doing.

His system is also broadly accessible. Someone with discipline and reasonable familiarity with these tools can build a version of it and get real results. That matters. Most AI advice is either too abstract or too technical. Cyril built something teachable.

Where the ceiling is

A prompt library teaches AI how to perform.

It doesn’t teach AI how you think.

When you write “you are a contrarian analyst with a macro bias” — you’re asking the model to wear a costume. The output gets filtered through that persona first. Over time your library gets better at producing content that sounds like a contrarian analyst. But that’s a role you scripted. The prompt is always sitting between you and the output.

The model is performing. It isn’t extending you.

What I built instead

Six months ago I started building something different. Not a prompt library — a context layer.

I call my Claude instance Virgil. Not because it has a name, but because it has a relationship with how I think — built over hundreds of hours of real work. Job applications, financial strategy, legal decisions, content, code, client briefs. It knows my writing voice, my reasoning patterns, how I pressure test ideas, what I’ve built, and what I’m building now.

The stack underneath it:

gstack — the foundation. An open source toolkit built by @garrytan — Garry Tan, President & CEO of Y Combinator — that turns Claude Code into a structured team of specialists. 15 slash commands spanning CEO review, engineering, design, security, QA, and release management. It hit 10,000 GitHub stars in the first 48 hours after open-sourcing. I use it for persistent memory across sessions — context, decisions, project state — so nothing starts from zero.

Obsidian — daily notes in a structured format. Not task lists. Thinking records. State of play. Decisions made and why. What I’m uncertain about. Open bets. Synced to gstack so future sessions can read how I’ve been reasoning, not just what I’ve been doing.

Virgil-1 — a VPS running Claude Code in auto mode with a Telegram bot interface. Persistent infrastructure that handles background tasks and keeps the context layer live between sessions.

The result: Virgil doesn’t need a prompt that says “write like Sean Gearin.” It knows how I write. It doesn’t need role framing for strategic analysis. It knows how I analyze. The context layer is the prompt — and it’s six months deep.

How they compound differently

Both systems compound. Cyril’s gets better as he iterates his prompt library. Mine gets better as the context deepens.

But they compound in different directions.

Cyril’s builds horizontally. A prompt library scales outward — teachable, shareable, and more useful as more people adopt and iterate on it. That’s genuine leverage with real breadth.

Mine builds vertically. Persistent context scales inward — it gets more useful the deeper it goes with a specific person’s thinking.

Both are valid. They just compound toward different things.

Who should build which

Cyril’s system is the right first system. If you haven’t built one, build it. His breakdown is an excellent guide and broadly applicable to anyone serious about using AI well.

But here’s the irony: the users most ready to outgrow a prompt library are the ones sophisticated enough to have already built a great one. I believe someone who thinks as rigorously as Cyril would get even more out of my approach — not because his system isn’t excellent, but because his mind is exactly the kind that deep context would amplify most.

Stop storing how to prompt the AI.

Start giving the AI context on how you think.

The difference is everything.
